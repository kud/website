The **jenkins** plugin wraps the
[`@kud/mcp-jenkins`](https://www.npmjs.com/package/@kud/mcp-jenkins) MCP server,
giving Claude full Jenkins control: inspect builds, stream console logs, trigger
pipelines, manage queues, and diagnose CI failures.

## What it bundles

| Skill          | Trigger                      | What it does                                                   |
| -------------- | ---------------------------- | -------------------------------------------------------------- |
| `/ci-diagnose` | `/ci-diagnose <job> [build]` | Fetch a failing build's console log and explain the root cause |
| `/build-watch` | `/build-watch <job>`         | Trigger a build and watch it to completion with live status    |

## Install

```
/plugin install jenkins@kud --scope project
```

Project scope is recommended so a whole team shares the same Jenkins config via
`.claude/settings.json`.

## Configuration

Set these environment variables so the server can reach your Jenkins instance:

```
MCP_JENKINS_URL=https://jenkins.example.com
MCP_JENKINS_USER=your-username
MCP_JENKINS_API_TOKEN=your-api-token
```
