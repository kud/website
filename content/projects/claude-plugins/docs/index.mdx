**claude-plugins** is a [Claude Code](https://claude.ai/code) plugin
marketplace. Each plugin bundles one of kud's **MCP servers** with its
**companion skills** into a single installable unit.

Install a plugin → get the MCP tools _and_ the slash commands in one shot. No
juggling config files, no separate skill installs.

## Plugins

| Plugin                                                         | What it does                                              |
| -------------------------------------------------------------- | --------------------------------------------------------- |
| [opencode](/projects/claude-plugins/docs/opencode)             | Query any opencode-supported model for a second opinion   |
| [github-copilot](/projects/claude-plugins/docs/github-copilot) | Query GitHub Copilot models via the official SDK          |
| [jenkins](/projects/claude-plugins/docs/jenkins)               | Inspect builds, stream logs, and diagnose CI failures     |
| [harness-fme](/projects/claude-plugins/docs/harness-fme)       | Inspect and control Harness FME feature flags             |
| [trakt](/projects/claude-plugins/docs/trakt)                   | Track what you're watching — check in, watchlist, history |
| [raindrop-io](/projects/claude-plugins/docs/raindrop-io)       | Search and save Raindrop.io bookmarks                     |
| [google-keep](/projects/claude-plugins/docs/google-keep)       | Capture notes and manage Google Keep checklists           |

## Install a plugin

### 1. Register the marketplace

```
/plugin marketplace add kud/claude-plugins
```

This registers the marketplace as `@kud`, so every plugin is referenced as
`<name>@kud`.

### 2. Install a plugin

Install at **user** scope to make a plugin available everywhere, or **project**
scope to share it with a team via `.claude/settings.json`:

```
/plugin install opencode@kud --scope user
/plugin install jenkins@kud --scope project
```

Each plugin page lists its own install command and any credentials it needs.

### 3. Set up credentials

Plugins that talk to authenticated services (Jenkins, Harness FME, Trakt,
Raindrop.io, Google Keep) need environment variables or a one-time setup
command. Each plugin page documents exactly what's required.

### 4. Use a skill

Once installed, the plugin's slash commands are available immediately:

```
/ask-opencode explain this function
/ci-diagnose my-service
/trakt-whats-on
```

## How it works

Each plugin lives in its own MCP server repo and carries its own manifest and
skills:

```
mcp-example/                  ← github.com/kud/mcp-example
├── .claude-plugin/
│   └── plugin.json           # MCP config + skill list
├── skills/
│   └── my-skill/
│       └── SKILL.md
└── src/
```

This marketplace is a thin index that bundles those servers with their skills so
a single `/plugin install` gives you both.

Pick a plugin from the list above to see what it bundles and how to install it.
