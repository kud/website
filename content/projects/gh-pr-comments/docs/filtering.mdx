---
title: Filtering
description: Narrow review threads by file, author, date, state, and sort order.
---

You can pre-filter threads when launching the tool, then refine further inside
the fzf picker.

## By file

The UI list starts with the file path for quick scanning. Filter by file with
`-f`, which can be repeated:

```sh
gh pr-comments -f src/app.py
```

Values are treated as regex by jq's `test(...)`, so you can use patterns:

```sh
gh pr-comments -f '^src/.*\.py$'
```

## By author

```sh
gh pr-comments -a @alice
```

`--author` accepts either `user` or `@user`, and can be repeated. `copilot` is
normalized to `copilot-pull-request-reviewer`.

## By date

Filter by an inclusive, UTC-based range on each thread's `createdAt`:

```sh
gh pr-comments --since 2024-01-01 --until 2024-01-31
```

## Combine filters

Filters stack together:

```sh
gh pr-comments -f src -a @alice --since 2024-01-01 --include-outdated
```

## Sorting and state

- Sort: `--sort date` sorts by newest first; `--sort author` groups by author.
  The default is `file`.
- Resolved: `--resolved` browses resolved threads as a follow-up queue instead
  of unresolved ones.

## Color

Disable colors with `--no-color`, or set `NO_COLOR=1`. Piping output? Prefer
`--no-color`.
