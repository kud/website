<Section
  eyebrow="The problem"
  title="Stop hand-rolling ANSI escapes"
  sub="Most Zsh scripts scatter raw escape codes everywhere — unreadable, inconsistent, impossible to theme. shui replaces them with a real design system: semantic components you compose, and tokens you restyle."
>
  <Install label="without shui">

```zsh
echo -e "\033[32m✓\033[0m Deployed"
echo -e "\033[1;31m✗\033[0m Build failed"
echo -e "\033[33m⚠\033[0m  Low disk space"
```

  </Install>

  <Install label="with shui">

```zsh
shui message success "Deployed"
shui message error   "Build failed"
shui message warning "Low disk space"
```

  </Install>
</Section>

<Section
  eyebrow="The system"
  title="A component for every part of your output"
  sub="Twenty-plus semantic components — messages, layout, inline tags, boxes, tables, progress, and interactive prompts — all sharing one consistent API. You compose your script's output instead of painting it by hand."
>
  <Install label="messages">

```console
$ shui message success "Deployed to production"
✅ Deployed to production
$ shui message error "Connection refused"
❌ Connection refused
$ shui message warning "API key expires in 3 days"
⚠️ API key expires in 3 days
```

  </Install>

  <Install label="tables">

```console
$ shui table "Package|Version|Status" "node|20.11.0|OK" "bun|1.1.3|OK"
┌─────────┬──────────┬────────┐
│ Package │ Version  │ Status │
├─────────┼──────────┼────────┤
│ node    │ 20.11.0  │ OK     │
│ bun     │ 1.1.3    │ OK     │
└─────────┴──────────┴────────┘
```

  </Install>

  <Install label="progress">

```console
$ shui progress 50 100 --label="Downloading "
Downloading ████████████████████░░░░░░░░░░░░░░░░░░░░ 50%
```

  </Install>

  <Install label="interactive prompts">

```console
$ shui confirm "Deploy to production?"
? Deploy to production? › (y/N)

$ shui select "Pick an environment" staging production
? Pick an environment ›
  ❯ staging
    production
```

  </Install>

<Section
  eyebrow="The theme engine"
  title="Tokens, not hardcoded colours"
  sub="Every component reads from a token layer — colours, icons, and styles set by environment variables. Swap a theme or an icon set and your whole output restyles, without touching a single component call. That is what makes shui a design system and not a bag of helpers."
>
  <Install label="restyle everything from the outside">

```zsh
SHUI_THEME=minimal     # default · minimal — swap the whole look
SHUI_ICONS=emoji       # nerd-font glyphs → emoji, instantly
NO_COLOR=1             # respected automatically, degrades to plain ASCII
```

  </Install>
</Section>

<Section
  eyebrow="Features"
  title="Built for real shell scripts"
  sub="Everything you need to give your scripts a consistent, themeable voice."
>
  <Features>
    <Feature title="Unified message API">
      One command for success, error, warning, info, and muted messages — no
      escape codes in sight.
    </Feature>
    <Feature title="Token-based themes">
      Swap colours, icons, and styles via environment variables, without
      touching component code.
    </Feature>
    <Feature title="Inline components">
      Badges and pills write to stdout without a newline, composing naturally
      inside command substitutions.
    </Feature>
    <Feature title="Interactive prompts">
      confirm, select, radio, multiselect, and input — keyboard navigation and
      sensible defaults included.
    </Feature>
    <Feature title="Progress & spinners">
      progress, spinner, and loader, with optional native iTerm2 dock badge
      integration.
    </Feature>
    <Feature title="Zero dependencies">
      A single source file, pure Zsh. No npm, no brew, nothing to install. And
      NO_COLOR aware out of the box.
    </Feature>
  </Features>
</Section>
